## huggingface 如何不被Sleep

Hugging Face 免费服务器的休眠规则是连续 48 小时无外部请求触发—— 这里的 “请求” 指从服务器外发起的 HTTP/HTTPS 请求（比如访问服务器部署的应用页面、调用 API 接口），内部进程运行不算。

![img](https://imgoss.xgss.net/picgo-tx2025/QQ_1761670621591.png?tx)

方案 1：新手首选「第三方监控工具」（零代码）

适合不懂代码的用户，用现成工具定时发请求，步骤仅 3 步：

1. **选工具**：推荐 UptimeRobot（免费版足够，这里以 UptimeRobot 为例；

1. **填服务器地址**：

- 打开 UptimeRobot 官网（uptimerobot.com）注册，点击「Add New Monitor」；

- 「Monitor Type」选「HTTP (s)」，「Friendly Name」填自定义名称（比如 “HuggingFace 保活”）；

- 「URL (or IP)」填你的 Hugging Face Space 访问地址（格式：https://[你的 Space 名称]-[用户名].hf.space）；

1. **设定时频率**：

- 「Check Interval」选「30 minutes」（每 30 分钟发 1 次请求，远低于 48 小时阈值）；

- 其他默认，点击「Create Monitor」，工具会自动定时发请求，服务器永不休眠。



不要使用v2流量工具，被封禁了！

## 需要注意

**休眠机制**：免费服务器 48 小时无请求会自动休眠，程序会暂停运行。解决办法：设置定时请求（比如用云函数发心跳包）。

**依赖兼容**：非 Python 应用建议用 Docker 部署，Hugging Face 对容器支持度拉满，环境兼容性问题少。

**流量限制**：虽标注 “无限流量”，但实测单 IP 单日请求超 1000 次可能被限流，个人测试完全够用。

**不要使用特殊软件**：使用某些特殊的软件会被封禁。
