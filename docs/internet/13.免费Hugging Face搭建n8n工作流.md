- 项目名称自定义即可
- 数据库密码务必妥善保存，后续配置需要用到

## 五、部署n8n核心服务

### 2.设置变量（重点）

Space创建后，进入Space 工作台，点击 Setting 设置环境变量，主要就是设置PostgreSQL数据库连接相关信息。

下拉，Variables and secrets选项

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1768901744594.png?tx)

Variables and secrets

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1768901797809.png?tx)



**环境变量配置是整个部署的核心环节：**

公开变量需要设置：

- DB_TYPE=postgresdb
- 数据库地址、端口、数据库名称
- N8N相关的基础配置，如端口、协议、访问地址等

公开环境变量名称有：

| 环境变量（Variable Name）             | 值（Value）                                                  | 说明                                                         |
| ------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| DB_TYPE                               | postgresdb                                                   | 告诉n8n我们使用的是PostgreSQL数据库。                        |
| DB_POSTGRESDB_HOST                    | [你的 Neon Host]     我填写：ep-odd-snow-ah0yh1g2-pooler.c-3.us-east-1.aws.neon.tech | 从 Neon 获取的数据库主机地址。                               |
| DB_POSTGRESDB_PORT                    | 5432                                                         | PostgreSQL的标准端口号是5432，具体看你的数据库连接信息。     |
| DB_POSTGRESDB_DATABASE                | neondb                                                       | 在 Neon 中使用的数据库名。                                   |
| N8N_PORT                              | 7860                                                         | n8n容器内部监听的端口。HuggingFace会自动将外部的443端口映射到这里。 |
| N8N_PROTOCOL                          | https                                                        | 告诉n8n它的公开访问协议是HTTPS。                             |
| N8N_EDITOR_BASE_URL                   | [你的Space URL]   我填写：https://star0098-n8n.hf.space      | 非常重要。n8n的完整公开访问地址,例如要带https！ ，YourName-YourSpaceName.hf.space,这能确保所有重定向、Webhook和OAuth回调都使用正确的URL. |
| DB_POSTGRESDB_SSL                     | true                                                         | 关键设置                                                     |
| N8N_ENFORCE_SETTINGS_FILE_PERMISSIONS | true                                                         | 解决日志中关于文件权限的警告，让 n8n自动管理其配置文件的权限。 |

**私密变量则包含敏感信息：**

- 数据库用户名和密码
- N8N加密密钥
- Webhook回调地址

私密环境变量名称有：

| 环境变量(Secret Name)  | 值(Value)                                               | 说明                                                         |
| ---------------------- | ------------------------------------------------------- | ------------------------------------------------------------ |
| DB_POSTGRESDB_USER     | [你的Neo用户名]                                         | Neon数据库的用户名                                           |
| DB_POSTGRESDB_PASSWORD | [你的Neo密码]                                           | 你在创建Neon的数据库密码。                                   |
| N8N_ENCRYPTION_KEY     | [随机生成的长字符串]                                    | 很重要。用于加密和解密n8n中存储的凭证(Credentilals)。请务必备份好这个值! |
| WEBHOOK_URL            | [你的Space URL] ，我填的：https://star0098-n8n.hf.space | 如 YourName-YourSpaceName.hf.space.与 N8N_EDITOR_BASE_URL 作用类似,用于确保Webhook能被正确触发。虽然有些冗余，但加上可以保证对旧版工作流的兼容性。 |

配置环境变量时要特别注意区分公开和私密的区别，涉及密码、密钥等信息必须设置为Secret。

### 3.Dockerfile配置与启动

在Space的Files页面创建Dockerfile，内容很简单：

```plaintext
FROM n8nio/n8n
```

## 常见问题排查

如果部署过程中遇到问题，首先检查环境变量配置是否正确，特别是数据库连接相关的参数。大多数失败案例都是因为数据库地址、端口或密码配置错误导致的。
