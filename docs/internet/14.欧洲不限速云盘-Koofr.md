## 二、关键前置：生成 WebDAV 应用密码（核心操作！）

Koofr 的灵活扩展性主要依赖 WebDAV 协议，无论是搭建图床还是联动其他工具，第一步都要先生成专属应用密码，这步操作不难，但有个关键点要记牢：

### 1.注册账号

注册并登录 Koofr 账号后，点击右上角头像，选择 “首选项” 进入设置界面；

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1769601303943.png?tx)

选择谷歌登录

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1769601358540.png?tx)





### 2.创建WebDAV密码

点击首选项、在左侧菜单中找到 “密码” 选项，下拉至 “应用密码” 板块 —— 这里的密码专门用于 WebDAV 等第三方应用访问，不能用来登录网页或移动客户端；

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1769601471976.png?tx)



输入应用名称（比如 “WebDAV”“图床搭建”），点击 “生成”，系统会自动生成 16 位字符的应用密码；

敲黑板！这个密码**只会显示一次**，生成后一定要及时复制保存，丢失后无法找回，只能重新生成并替换原有配置。

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1769655064791.png?tx)

### 3.Koofr的WebDAV的连接参数

可以用任何支持WebDAV的工具去连接。

```
https://app.koofr.net/dav/Koofr

用户名：注册Koofr用的邮箱 (谷歌的账号邮箱)

密码：刚生成的应用密码
```

也可以使用RaiDrive（windows软件）、FolderSync等工具

## 服务器端：搭建 WebDAV 图床（适合创作者 / 站长）

如果需要将 Koofr 作为图床，用于博客、网站图片存储，推荐用宝塔面板 + Docker+Filestash 实现，步骤简单易操作：

1. 登录宝塔面板，在 Docker 商店中搜索 “filestash”，找到对应应用后直接安装，默认运行端口为 8334（可自行修改）；
2. 安装完成后，通过 “服务器 IP+8334 端口” 访问 Filestash，先设置后台管理密码，然后退出登录，进入连接配置界面；
3. 填写 Koofr 的 WebDAV 连接参数：地址填`https://app.koofr.net/dav/Koofr`（端口默认 443），用户名是注册 Koofr 时用的邮箱，密码就是刚才生成的 16 位应用密码；
4. 点击 “连接” 后，即可成功关联 Koofr 网盘，上传一张图片测试，能正常生成访问链接，图床搭建就完成了！后续网站、博客直接调用该链接即可，稳定性实测靠谱。
