## 🛠️ 零成本部署全流程（5步走）

跟着星哥的清单，小白也能一次跑通：

### 第一步：注册 Cloudflare 账号

1. 打开 https://dash.cloudflare.com/sign-up
2. 输入邮箱和密码，完成注册

### 第二步：创建 Worker

1. 登录后点击左侧菜单「Workers & Pages」
2. 点击「Create」→「Create Worker」
3. 名称填 `movecar`（或你喜欢的名字）
4. 点击「Deploy」
5. 点击「Edit code」，删除默认代码
6. 复制 `movecar.js` 全部内容粘贴进去
7. 点击右上角「Deploy」保存

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1780330388442.png?tx)

### 第三步：创建 KV 存储

1. 左侧菜单点击「KV」
2. 点击「Create a namespace」
3. 名称填 `MOVE_CAR_STATUS`，点击「Add」
4. 回到你的 Worker →「Settings」→「Bindings」
5. 点击「Add」→「KV Namespace」
6. Variable name 填 `MOVE_CAR_STATUS`
7. 选择刚创建的 namespace，点击「Deploy」



### 第四步：配置环境变量

1. Worker →「Settings」→「Variables and Secrets」
2. 添加以下变量：
   - `BARK_URL`：你的 Bark 推送地址（如 `https://api.day.app/xxxxx`）
   - `PHONE_NUMBER`：备用联系电话（可选）

### 第五步：绑定域名（可选）

1. Worker →「Settings」→「Domains & Routes」
2. 点击「Add」→「Custom Domain」
3. 输入你的域名，按提示完成 DNS 配置



### 生成颜值二维码

- 拿到最终可访问的链接，使用草料二维码/联图网进行美化（加圆角、换底色、印车牌号）。
- 打印成 PVC 贴纸放车内，完工！



## 实战避坑指南

| 常见问题                    | 原因分析                                    | 星哥解法                                              |
| :-------------------------- | :------------------------------------------ | :---------------------------------------------------- |
| 🔒 微信扫码提示“已停止访问”  | 微信对 `workers.dev` 及部分免费域名风控极严 | 换用 `cloudns.ph` + CF 托管 + CNAME，避开黑名单后缀   |
| 📤 推送请求发不出去          | 原版 `GET` 请求易被运营商或平台拦截         | 强制改为 `POST`，并添加标准 `User-Agent` 头           |
| 📍 不弹定位权限/位置获取失败 | 非 HTTPS 环境或域名未正确解析               | 确保 CF 开启 `Full SSL`，浏览器强制走 `https://`      |
| 📱 微信内置浏览器跳转异常    | 微信 WebView 安全策略限制                   | 提示用户“长按图片识别二维码”或改用系统相机/支付宝扫码 |
