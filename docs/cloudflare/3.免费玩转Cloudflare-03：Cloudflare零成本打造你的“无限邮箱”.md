## 方案一：自建临时邮箱网站

如果你希望拥有一个功能完整、支持在线收发、带附件管理，甚至能开放给他人使用的临时邮箱站点，开源项目 `cloudflare_temp_email` 是目前的最佳选择。

### 🔧 核心优势

- 独立网页端，支持多设备访问
- 自动过期清理，保护隐私不泄露
- 完全基于 Cloudflare 免费生态（Pages + Worker + D1）

### 📝 部署路径

1. 访问 GitHub 仓库：`dreamhunter2333/cloudflare_temp_email`
2. 按照官方 README 的指引，依次完成 D1 数据库绑定、Worker 部署、Pages 前端上线。
3. 绑定你自己的域名，即可生成专属临时邮箱服务。

## 方案二：Cloudflare邮件转发

不想写代码、不想管服务器？只想安静地收邮件？Cloudflare 自带的**电子邮件路由（Email Routing）**功能就是为你量身定做的。

它的逻辑非常纯粹：把所有发往 `@你的域名` 的邮件，统一转发到你常用的 Gmail、Outlook 或 QQ 邮箱。

四步完成配置

### ① 开启邮件路由功能

登录 Cloudflare 控制台，进入你的域名管理页 → 左侧菜单找到 **`电子邮件服务` → `电子邮件路由`**，点击开启。

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1779294682357.png?tx)

### ② 自动补全 DNS 记录

首次启用时，Cloudflare 会提示你添加必要的解析记录：

- **MX 记录**：指引外部邮件服务器将信件投递至 Cloudflare

- TXT 记录：配置 SPF 验证，防伪造、提升进件率

  ![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1779294724475.png?tx)

  

  查看域名解析新增了几条记录

  ![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1779295165780.png?tx)

  > 💡 直接点击页面上的“添加记录并启用”即可。
  >
  > **注意**：若该域名此前绑定过其他企业邮箱或免费邮，请先清理冲突的旧 MX 记录，否则转发会失效。

  

### ③ 创建专属前缀 & 设置转发

在“路由规则”面板中点击“创建地址”：

- **自定义地址**：填写你喜欢的别名，例如 `shop`、`star`、`vip`
- **目标地址**：填入你日常使用的主力邮箱 保存后，一个专属的域名邮箱就诞生了。

目标地址：即你最终收到的邮件地址

Cloudflare 会向你的目标邮箱发送一封确认信。登录该邮箱，点击验证链接，状态变为“已验证”后，转发链路正式打通。

| 维度           | 方案一：自建临时邮箱站           | 方案二：原生邮件转发         |

## 星哥实战避坑 & 进阶 Tips

1. **强烈推荐开启“全部捕获（Catch-all）”**
   在路由规则底部，将“全部捕获地址”设置为你的主邮箱。开启后，无论对方发到 `test@`、`random123@` 还是 `info@`，邮件都会自动汇聚到你的收件箱。配合邮箱的标签/过滤规则，分类管理效率直接拉满。
2. **别名命名法则**
   建议采用 `用途+日期` 或 `平台名+随机串` 的格式，例如 `taobao_0520@`、`bilibili_sub@`。一旦某个渠道开始滥发广告，直接删除该别名规则，源头切断，干净利落。
3. **发件限制说明**
   Cloudflare 邮件路由目前**仅支持收信转发**。若你需要以 `@yourdomain.com` 的身份对外发信，建议搭配 SMTP 服务（如 Resend、SendGrid 或自建邮局）使用。
