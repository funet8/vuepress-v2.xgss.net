# 数据库性能监控：利用 Prometheus 工具监控性能

数据库性能监控是保障业务稳定性与可扩展性的核心环节。随着微服务与云原生架构的普及，Prometheus 已成为最主流的监控方案之一，凭借其高效的数据采集、灵活的查询语言（PromQL）以及与 Grafana 的天然集成，成为数据库监控的首选工具。

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1768809254701.png?tx)

------

## 🧩 一、为什么选择 Prometheus 监控数据库

### ⭐ Prometheus 的优势

- **拉取式采集模型（Pull）**：避免被动推送带来的不可控性。
- **多维度数据模型**：适合复杂数据库指标分析。
- **PromQL 强大查询能力**：可轻松构建延迟、QPS、慢查询等指标。
- **生态丰富**：几乎所有主流数据库都有成熟 Exporter。
- **与 Grafana 深度集成**：可视化能力强，适合运维与开发团队协作。

------

## 🏗️ 二、数据库监控架构示意

```
Database → Exporter → Prometheus → Grafana → 告警系统(Alertmanager)
```

- **Exporter**：负责采集数据库内部指标并暴露为 HTTP 接口。
- **Prometheus**：定期拉取 Exporter 数据并存储。
- **Grafana**：展示监控图表。
- **Alertmanager**：处理告警规则。

------

## 🛠️ 三、常见数据库 Exporter 选择

| 数据库类型 | Exporter 名称     | 说明                       |
| ---------- | ----------------- | -------------------------- |
| MySQL      | mysqld_exporter   | 官方维护，指标最丰富       |
| PostgreSQL | postgres_exporter | 支持自定义 SQL 监控        |
| Redis      | redis_exporter    | 支持内存、命中率、慢查询等 |
| MongoDB    | mongodb_exporter  | 支持连接数、锁、oplog 等   |
| Oracle     | oracledb_exporter | 社区维护，指标较全         |



## 四、以 Redis 为例：部署 Exporter

### 1. 下载并运行 redis_exporter

```bash
docker run -d \
  --name redis_exporter \
  -p 9121:9121 \
  oliver006/redis_exporter
```

### 2. 配置 Prometheus 抓取 Redis 指标

编辑 `prometheus.yml`：

```yaml
scrape_configs:
  - job_name: 'redis'
    static_configs:
      - targets: ['localhost:9121']
```

重启 Prometheus 后即可开始采集。

## 五、关键数据库性能指标（通用）

### 1. 性能指标

- **QPS / TPS**：每秒查询/事务数
- **查询延迟（latency）**
- **慢查询数量**
- **连接数与连接池使用率**

### 2. 资源指标

- **CPU 使用率**
- **内存占用**
- **磁盘 IOPS / 吞吐**
- **网络流量**

### 3. 数据库特有指标（示例）

- Redis：命中率、内存碎片率、阻塞命令
- MySQL：InnoDB Buffer Pool 命中率、锁等待
- PostgreSQL：WAL 写入、Vacuum 状态

## 🔍 六、PromQL 常用查询示例

### 1. QPS（每秒查询数）

```promql
rate(redis_commands_processed_total[1m])
```

### 2. 内存使用

```promql
redis_memory_used_bytes
```

### 3. 连接数

```promql
redis_connected_clients
```

### 4. 慢查询

```promql
rate(redis_slowlog_count_total[5m])
```

## 🚨 八、告警规则示例（Alertmanager）

### Redis 内存使用超过 80%

```yaml
- alert: RedisMemoryHigh
  expr: redis_memory_used_bytes / redis_memory_max_bytes > 0.8
  for: 5m
  labels:
    severity: warning
  annotations:
    summary: "Redis 内存使用率过高"
```

## 🧭 九、最佳实践

- **Exporter 与数据库部署在同一台机器**，避免网络延迟影响采集。
- **Prometheus 分片或联邦集群**，应对大规模数据库监控。
- **结合日志系统（ELK / Loki）**，实现全链路分析。
- **定期清理 Prometheus 数据**，避免存储膨胀。
- **监控 + 告警 + 自动化运维联动**，提升系统自愈能力。
