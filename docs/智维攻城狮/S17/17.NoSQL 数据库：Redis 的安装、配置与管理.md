# NoSQL 数据库：Redis 的安装、配置与管理

Redis（Remote Dictionary Server）是一款开源、基于内存的高性能 NoSQL 数据库，广泛应用于缓存、会话管理、消息队列、实时统计等场景。其核心优势包括极高的读写性能、丰富的数据结构、持久化能力以及灵活的集群方案。

本文将从 **安装、配置、服务管理、权限控制、持久化、性能优化** 六个维度，构建一套可直接落地的 Redis 管理指南。

------

# **一、Redis 的安装**

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1768274733961.png?tx)

## 🧩 1. Linux 安装（推荐生产环境）

### **方式 A：使用包管理器安装（Ubuntu/Debian）**

```bash
sudo apt update
sudo apt install redis-server
```

### **方式 B：使用包管理器安装（CentOS/RHEL）**

```bash
sudo yum install redis
```

### **方式 C：源码编译安装（适合自定义优化）**

```bash
wget https://download.redis.io/redis-stable.tar.gz
tar -xzf redis-stable.tar.gz
cd redis-stable
make
sudo make install
```

------

## 🧩 2. macOS 安装（Homebrew）

```bash
brew install redis
brew services start redis
```

------

## 🧩 3. Windows 安装（WSL 或 Docker）

Redis 官方不再维护 Windows 版本，推荐使用：

### **方式 A：WSL**

```bash
sudo apt install redis-server
```

### **方式 B：Docker**

```bash
docker run -d --name redis \
  -p 6379:6379 \
  redis:latest
```

# **二、Redis 的核心配置（redis.conf）**

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1768274759979.png?tx)

Redis 的主要配置集中在 `redis.conf` 文件中，以下为常用关键项：

## 🔧 1. 基础配置

| 配置项                       | 说明                           |
| ---------------------------- | ------------------------------ |
| `bind 127.0.0.1`             | 限制监听地址，生产环境必须开启 |
| `port 6379`                  | 服务端口                       |
| `daemonize yes`              | 后台运行                       |
| `logfile /var/log/redis.log` | 日志路径                       |
| `databases 16`               | 数据库数量                     |

------

## 🔐 2. 安全配置

| 配置项                       | 说明                     |
| ---------------------------- | ------------------------ |
| `requirepass <password>`     | 设置访问密码             |
| `rename-command FLUSHALL ""` | 禁用危险命令             |
| `protected-mode yes`         | 默认开启，防止未授权访问 |

------

## 💾 3. 持久化配置（RDB/AOF）

### **RDB 快照**

```conf
save 900 1
save 300 10
save 60 10000
```

### **AOF 日志**

```conf
appendonly yes
appendfsync everysec
```

------

## 🚀 4. 性能相关配置

| 配置项                         | 说明                  |
| ------------------------------ | --------------------- |
| `maxmemory 2gb`                | 最大内存限制          |
| `maxmemory-policy allkeys-lru` | 淘汰策略              |
| `tcp-backlog 511`              | 高并发优化            |
| `io-threads 4`                 | 多线程 IO（Redis 6+） |

# **三、Redis 服务管理**

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1768274781263.png?tx)

## ▶️ 启动 Redis

```bash
redis-server /etc/redis/redis.conf
```

## 🛑 停止 Redis

```bash
redis-cli shutdown
```

## 🔄 重启 Redis（systemd）

```bash
sudo systemctl restart redis
```

## 📊 查看状态

```bash
sudo systemctl status redis
```

# **四、用户与权限管理（Redis 6+ ACL）**

Redis 6 引入 ACL（Access Control List），支持多用户与细粒度权限。

## 👤 1. 创建用户

```bash
ACL SETUSER appuser on >password ~* +@all
```

## 🔒 2. 限制用户权限

```bash
ACL SETUSER readonly on >password ~* +get +exists -set -del
```

## 📋 3. 查看用户权限

```bash
ACL LIST
```

# **五、Redis 持久化与备份**

## 💾 1. RDB 备份

RDB 文件默认位于：

```
/var/lib/redis/dump.rdb
```

可直接复制备份。

## 🧾 2. AOF 备份

```
/var/lib/redis/appendonly.aof
```

## 🔄 3. 恢复数据

将 RDB/AOF 文件放回数据目录后重启 Redis 即可。

# **六、性能优化与最佳实践**

## ⚡ 1. 内存优化

- 使用 `maxmemory` 限制内存
- 合理选择淘汰策略（LRU/LFU）
- 避免存储大 Key（>1MB）

## 🧱 2. 网络与连接优化

- 开启 `tcp-keepalive`
- 使用连接池（如 JedisPool、Lettuce）

## 🧵 3. 多线程优化（Redis 6+）

```conf
io-threads 4
io-threads-do-reads yes
```

## 🗂 4. 数据结构选择

| 场景     | 推荐结构       |
| -------- | -------------- |
| 排行榜   | Sorted Set     |
| 会话缓存 | String         |
| 标签系统 | Set            |
| 计数器   | String（INCR） |
| 消息队列 | List / Stream  |
