# NoSQL 数据库：MongoDB 的安装、配置与管理

MongoDB 作为当前最流行的 NoSQL 文档型数据库之一，以其灵活的数据模型、水平扩展能力和丰富的生态工具，被广泛应用于互联网业务、日志系统、实时分析等场景。本文从安装、配置到管理运维，提供一套完整的 MongoDB 使用指南，适合开发者、运维工程师和架构师快速上手并构建稳定的数据库环境。

# **一、MongoDB 概述**

## **1.1 什么是 MongoDB**

MongoDB 是一个基于文档存储的 NoSQL 数据库，数据以 BSON（二进制 JSON）格式存储，天然支持灵活的结构和快速迭代。

**核心特点：**

- **Schema-Free**：无需固定表结构，适合快速变化的业务
- **高可扩展性**：支持分片（Sharding）实现水平扩展
- **高可用性**：Replica Set 提供自动故障切换
- **丰富的查询能力**：支持索引、聚合、全文搜索等
- **生态完善**：Compass、Atlas、驱动、监控工具齐全

# **二、MongoDB 的安装**

以下以主流平台为例，提供标准安装流程。

------

## **2.1 Linux（Ubuntu）安装**

### **步骤 1：导入官方 GPG Key**

```bash
curl -fsSL https://pgp.mongodb.com/server-7.0.asc | sudo gpg -o /usr/share/keyrings/mongodb-server-7.0.gpg --dearmor
```

### **步骤 2：添加官方源**

```bash
echo "deb [ signed-by=/usr/share/keyrings/mongodb-server-7.0.gpg ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list
```

### **步骤 3：安装 MongoDB**

```bash
sudo apt update
sudo apt install -y mongodb-org
```

### **步骤 4：启动服务**

```bash
sudo systemctl start mongod
sudo systemctl enable mongod
```

------

## **2.2 macOS（Homebrew）安装**

```bash
brew tap mongodb/brew
brew install mongodb-community@7.0
brew services start mongodb-community@7.0
```

------

## **2.3 Windows 安装（MSI）**

1. 访问 MongoDB 官网下载 MSI 安装包
2. 勾选 **Install MongoDB as a Service**
3. 默认端口 27017
4. 安装完成后可通过 `services.msc` 管理服务

# **三、MongoDB 基础配置**

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1768274621634.png?tx)

MongoDB 的核心配置文件通常位于：

- Linux: `/etc/mongod.conf`
- macOS: `/usr/local/etc/mongod.conf`
- Windows: `C:\Program Files\MongoDB\Server\7.0\bin\mongod.cfg`

------

## **3.1 常用配置项说明**

### **监听地址**

```yaml
net:
  bindIp: 127.0.0.1,192.168.1.10
  port: 27017
```

### **数据与日志路径**

```yaml
storage:
  dbPath: /var/lib/mongo
systemLog:
  path: /var/log/mongodb/mongod.log
```

### **启用认证**

```yaml
security:
  authorization: enabled
```

### **副本集配置**

```yaml
replication:
  replSetName: rs0
```

# **四、MongoDB 用户与权限管理**

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1768274649481.png?tx)

## **4.1 创建管理员用户**

进入 Mongo Shell：

```bash
mongosh
```

创建用户：

```javascript
use admin
db.createUser({
  user: "root",
  pwd: "StrongPassword",
  roles: ["root"]
})
```

------

## **4.2 常用角色**

| 角色         | 权限说明   |
| ------------ | ---------- |
| read         | 只读       |
| readWrite    | 读写       |
| dbAdmin      | 管理数据库 |

# **六、备份与恢复**

MongoDB 提供 `mongodump` 和 `mongorestore` 工具用于逻辑备份。

------

## **6.1 备份数据库**

```bash
mongodump --out /backup/mongo-202401/
```

备份指定库：

```bash
mongodump -d mydb -o /backup/
```

------

## **6.2 恢复数据库**

```bash
mongorestore /backup/mongo-202401/
```

恢复指定库：

```bash
mongorestore -d mydb /backup/mydb/
```

# **七、性能优化与最佳实践**

## **7.1 索引优化**

常用索引：

```javascript
db.users.createIndex({ email: 1 })
db.logs.createIndex({ created_at: -1 })
```

建议：

- 为高频查询字段建立索引
- 避免过多索引导致写入变慢
- 使用复合索引优化多条件查询

------

## **7.2 存储优化**

- 使用 WiredTiger（默认）提升并发性能
- 合理设置 `cacheSizeGB`
- 使用分片提升大规模数据吞吐

------

## **7.3 副本集与高可用**

副本集配置示例：

```javascript
rs.initiate({
  _id: "rs0",
  members: [
    { _id: 0, host: "mongo1:27017" },
    { _id: 1, host: "mongo2:27017" },
    { _id: 2, host: "mongo3:27017" }
  ]
})
```
