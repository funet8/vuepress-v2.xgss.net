# 性能分析工具：使用 sysstat 分析性能问题

在 Linux 运维与系统优化领域，性能分析工具是定位瓶颈、提升系统稳定性的关键手段。**sysstat** 是最常用、最成熟的系统性能监控工具集之一，涵盖 CPU、内存、磁盘 I/O、网络、进程等多个维度，能够提供实时监控与历史数据分析能力。

本文将系统介绍 sysstat 的核心组件、使用方法以及典型场景，帮助你快速掌握该工具并应用于实际生产环境。

------

## 1. sysstat 是什么？

**sysstat** 是 Linux 上最经典的性能监控工具集，由多个命令组成，覆盖系统运行的关键指标。它的特点包括：

- **轻量级**：资源占用极低，可长期运行
- **功能全面**：CPU、I/O、内存、网络、进程全覆盖
- **支持历史记录**：可记录长期性能数据
- **可视化友好**：支持 CSV/JSON 输出，便于接入 Grafana、Prometheus

sysstat 工具集主要包含以下组件：

| 工具          | 功能说明                                         |
| ------------- | ------------------------------------------------ |
| **sar**       | 综合性能采集与历史分析（CPU、内存、I/O、网络等） |
| **iostat**    | 磁盘 I/O 性能分析                                |
| **mpstat**    | 多核 CPU 性能分析                                |
| **pidstat**   | 进程级 CPU、内存、I/O 分析                       |
| **nfsiostat** | NFS 性能监控                                     |
| **tapestat**  | 磁带设备性能监控                                 |

------

## 2. 安装 sysstat

### Debian/Ubuntu

```bash
sudo apt update
sudo apt install sysstat -y
```

### CentOS/RHEL

```bash
sudo yum install sysstat -y
```

### 启用数据采集（非常重要）

部分系统默认关闭 sysstat 数据采集，需要手动开启：

```bash
sudo sed -i 's/ENABLED="false"/ENABLED="true"/' /etc/default/sysstat
sudo systemctl enable --now sysstat
```

------

## 3. sysstat 核心工具与使用示例

------

## 3.1 CPU 性能分析：mpstat / sar -u

### 实时查看 CPU 使用情况

```bash
mpstat 1
```

关键字段：

- **%usr**：用户态 CPU
- **%sys**：内核态 CPU
- **%iowait**：等待 I/O
- **%irq / %soft**：硬中断/软中断
- **%steal**：虚拟化抢占

### 历史 CPU 使用情况

```bash
sar -u 1 5
```

常见分析场景：

- %iowait 高 → I/O 瓶颈
- %sys 高 → 内核调用频繁
- %steal 高 → 云主机资源争抢

------

## 3.2 内存分析：sar -r / sar -W

### 实时内存状态

```bash
sar -r 1
```

关键字段：

- **kbmemfree**：空闲内存
- **kbmemused**：已用内存
- **kbbuffers / kbcached**：缓存
- **kbcommit**：内存承诺量

### Swap 使用情况

```bash
sar -W 1
```

常见分析场景：

- swap 持续增长 → 内存不足
- commit 超过物理内存 → 存在 OOM 风险

------

## 3.3 磁盘 I/O 分析：iostat / sar -d

### 查看磁盘 I/O

```bash
iostat -x 1
```

关键字段：

- **r/s / w/s**：读写次数
- **await**：平均等待时间
- **svctm**：服务时间
- **%util**：磁盘利用率

### 历史 I/O 数据

```bash
sar -d 1 5
```

常见分析场景：

- await 高 → I/O 延迟严重
- util 接近 100% → 磁盘饱和
- r/s 或 w/s 异常高 → 应用 I/O 风暴

------

## 3.4 网络分析：sar -n

### 查看网络吞吐

```bash
sar -n DEV 1
```

关键字段：

- **rxpck/s / txpck/s**：收发包速率
- **rxkB/s / txkB/s**：收发字节速率
- **ifutil**：网卡利用率

### TCP 状态分析

```bash
sar -n TCP,ETCP 1
```

可用于排查：

- 连接失败
- 重传
- 拥塞

------

## 3.5 进程级分析：pidstat

### 按 CPU 排序

```bash
pidstat -u 1
```

### 查看进程 I/O

```bash
pidstat -d 1
```

### 查看进程上下文切换

```bash
pidstat -w 1
```

常见场景：

- 某进程 CPU 飙高
- I/O 读写异常
- 上下文切换频繁（锁竞争）

------

## 4. sysstat 的典型使用场景

------

### 场景 1：CPU 占用异常

```bash
mpstat 1
pidstat -u 1
```

定位高 CPU 进程 → 分析线程 → 排查锁竞争或死循环。

------

### 场景 2：系统卡顿、响应慢

```bash
iostat -x 1
sar -r 1
```

判断是 I/O 饱和还是内存不足。

------

### 场景 3：网络吞吐下降

```bash
sar -n DEV 1
sar -n TCP,ETCP 1
```

检查是否丢包、重传、带宽耗尽。

------

### 场景 4：历史性能回溯

sysstat 会自动记录历史数据：

```bash
sar -u -f /var/log/sysstat/sa10
```

可用于：

- 故障后回溯
- 性能趋势分析
- 容量规划

------

## 5. 最佳实践与优化建议

- **开启 sysstat 数据采集**（默认可能关闭）
- **结合 pidstat 定位进程级问题**
- **关注趋势而非瞬时值**
- **与 Grafana/Prometheus 集成**
- **定期采样（1 秒）更利于分析**
- **长期保存 sar 数据用于容量规划**

------

## 6. 总结

sysstat 是 Linux 系统中最强大、最全面的性能分析工具集之一。通过合理使用 sar、iostat、mpstat、pidstat 等工具，你可以快速定位系统瓶颈、提升服务稳定性，并为容量规划提供可靠的数据支撑。
