# PostgreSQL 管理：使用与优化 PostgreSQL 的技巧

本文从 **使用技巧、性能优化、配置调优、备份恢复、监控与日常维护** 五个维度，系统总结 PostgreSQL 管理的核心方法，帮助你快速构建稳定、高性能的数据库环境。

# **1. PostgreSQL 使用技巧**

## **1.1 基础操作与常用命令**

- 连接数据库

  ```bash
  psql -h 127.0.0.1 -U postgres -d mydb
  ```

- 查看数据库列表
   `\l`

- 查看表结构
   `\d table_name`

- 切换数据库
   `\c dbname`

- 导出 SQL

  ```bash
  pg_dump -U postgres mydb > backup.sql
  ```

## **1.2 常用 SQL 技巧**

- 使用 

  ```
  RETURNING
  ```

   获取插入/更新后的数据

  ```sql
  INSERT INTO users(name) VALUES('Tom') RETURNING id;
  ```

- 使用 

  ```
  UPSERT
  ```

  （ON CONFLICT）

  ```sql
  INSERT INTO users(id, name)
  VALUES (1, 'Tom')
  ON CONFLICT(id) DO UPDATE SET name = EXCLUDED.name;
  ```

- JSON 字段操作

  ```sql
  SELECT data->>'name' FROM users;
  ```

## **1.3 事务与锁管理**

- 查看锁

  ```sql
  SELECT * FROM pg_locks;
  ```

- 强制终止阻塞会话

  ```sql
  SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE pid = xxx;
  ```

# **2. PostgreSQL 性能优化技巧**

## **2.1 索引优化**

- 常用索引类型

  | 类型   | 场景               |
  | ------ | ------------------ |
  | B-Tree | 精确匹配、范围查询 |
  | Hash   | 等值查询           |
  | GIN    | JSONB、全文搜索    |
  | GiST   | 地理空间、模糊匹配 |
  | BRIN   | 大表、顺序数据     |

- 创建索引

  ```sql
  CREATE INDEX idx_user_name ON users(name);
  ```

- 避免过度索引：索引越多，写入越慢。

## **2.2 查询优化**

- 使用 

  ```
  EXPLAIN ANALYZE
  ```

   分析执行计划

  ```sql
  EXPLAIN ANALYZE SELECT * FROM users WHERE name='Tom';
  ```

- 避免 SELECT *，减少 I/O

- 使用分页优化（避免 OFFSET 大量跳过）

  ```sql
  SELECT * FROM table WHERE id > last_id LIMIT 50;
  ```

## **2.3 表结构优化**

- 使用 JSONB 替代 JSON（支持索引）

- 使用 PARTITION 分区大表

  ```sql
  CREATE TABLE logs (
    id serial,
    created_at date
  ) PARTITION BY RANGE (created_at);
  ```

# **3. PostgreSQL 配置调优**

## **3.1 核心配置项（postgresql.conf）**

| 参数                   | 作用                | 建议                       |
| ---------------------- | ------------------- | -------------------------- |
| `shared_buffers`       | 内存缓存            | 设置为内存的 25%           |
| `work_mem`             | 单次排序/哈希内存   | 2MB–64MB，按查询复杂度调整 |
| `maintenance_work_mem` | VACUUM/索引重建内存 | 512MB–2GB                  |
| `effective_cache_size` | 估算可用缓存        | 内存的 50%–75%             |
| `max_connections`      | 最大连接数          | 建议配合连接池             |

## **3.2 WAL 日志优化**

- 开启异步提交（提高写入性能）

  ```sql
  SET synchronous_commit = off;
  ```

- 调整 WAL 大小
   `wal_buffers = 16MB`

## **3.3 连接池优化**

推荐使用：

- **PgBouncer**（轻量、适合 OLTP）
- **Pgpool-II**（负载均衡、读写分离）

# **4. PostgreSQL 备份与恢复**

## **4.1 逻辑备份（pg_dump）**

适合小型数据库、迁移、结构备份

```bash
pg_dump -U postgres mydb > backup.sql
```

恢复：

```bash
psql -U postgres mydb < backup.sql
```

## **4.2 物理备份（pg_basebackup）**

适合大规模数据库、主从复制

```bash
pg_basebackup -D /data/backup -Fp -Xs -P -U repl
```

## **4.3 PITR（时间点恢复）**

- 开启 WAL 归档

  ```conf
  archive_mode = on
  archive_command = 'cp %p /archive/%f'
  ```

- 恢复到指定时间

  ```conf
  recovery_target_time = '2026-01-13 10:00:00'
  ```

# **5. PostgreSQL 监控与日常维护**

## **5.1 常用监控视图**

- 活跃连接

  ```sql
  SELECT * FROM pg_stat_activity;
  ```

- 索引使用情况

  ```sql
  SELECT * FROM pg_stat_user_indexes;
  ```

- 表膨胀检查

  ```sql
  SELECT * FROM pgstattuple('table_name');
  ```

## **5.2 VACUUM 与自动清理**

- 手动清理

  ```sql
  VACUUM ANALYZE;
  ```

- 自动清理参数

  | 参数                           | 说明     |
  | ------------------------------ | -------- |
  | autovacuum                     | 是否开启 |
  | autovacuum_vacuum_scale_factor | 触发比例 |
  | autovacuum_naptime             | 执行间隔 |

## **5.3 日志分析**

- 开启慢查询日志

  ```conf
  log_min_duration_statement = 500ms
  ```
