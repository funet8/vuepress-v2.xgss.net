# 别再手动操作了！OpenClaw实例教程：腾讯云+阿里云，每天自动提醒，杜绝服务中断



## 需求

在日常运维或个人使用中，腾讯云、阿里云账号余额不足会导致服务中断、资源释放，尤其是依赖云服务器、云存储的业务，一旦余额告警不及时，可能造成数据丢失或业务停摆。

工作中管理阿里云、腾讯云账号有几十个，每天（每周）查看云平台的余额、显示每日金额的消耗、避免出现余额不足导致业务出现问题。

1.查看阿里云后台的余额

2.查看腾讯云后台的余额

本篇文章只是显示云账户余额，有些云服务重要的指标也可以按照此方法延伸一些功能，节约人力，真正雇一个AI牛马替你干活。

## 工作流

一、给阿里云、腾讯云等新建一个子账户，权限只有财务权限，供openclaw调用

二、给openclaw提交一个账号、密码登录，AccessKey ID和AccessKey Secret

三、每日查询云账户中的余额，（可以用浏览器，也可以使用官方接口）

四、将余额导入到Excel格式中，并且发送到我的邮箱

五、如果需要发送邮箱，需要提前准备SMTP授权码，这个用163的邮箱即可。

## 安全提醒

风险问题：可能泄露云服务的ID和KEY、邮箱的SMTP授权码，被坏人拿走了

openclaw的安全提醒： 为了安全起见，建议你更换一下 SecretKey，这次的 key 已经在服务器日志里暴露了。可以在腾讯云控制台 → 访问管理 → API密钥管理 创建新的并删除旧的。

问客服能不能IP白名单：新建子账户给到openclaw，能不能配置IP白名单，只有这个ip才能使用apikey
