# Openclaw开放公网



希望局域网的其他小伙伴也可以访问你的openclaw小龙虾。

```
局域网访问：http://我的IP:18789?token=我的Token
```



想要把 OpenClaw 的绑定地址从仅本地访问的 `127.0.0.1` 改成允许公网访问的 `0.0.0.0`，我会给你**两种配置方式**（临时生效 / 永久生效），并重点提醒公网开放的安全注意事项。

## 环境

Windows11安装了openclaw

IP是：192.168.152.30



## 启用 Token 认证

以管理员身份运行 PowerShell，执行全套配置命令

### 1.设置绑定模式为lan（允许公网/局域网访问）

```
openclaw config set gateway.bind "lan"
```

### 2. 设置监听端口为18789

```
openclaw config set gateway.port 18789
```



### 3.自动生成Token

```
openclaw config unset gateway.auth.token
```



去配置文件中查看

```
    "auth": {
      "mode": "token",
      "token": "你的token"
    },
```



### 4.修改controlUi

用记事本 / VS Code 打开 `config.json`；

找到 `gateway` 节点，添加 / 修改 `controlUi` 配置（确保是 JSON 数组格式）：

```
{
  "gateway": {
    "bind": "lan",
    "port": 18789,
    "auth": {
      "mode": "token",
      "token": ""  // 留空，重启后自动生成
    },
    "controlUi": {
      "allowedOrigins": ["*"]  // 重点：数组格式 ["*"]
    }
  },
  "plugins": {
    "allow": ["qqbot"]
  }
}
```

保存文件（确保格式是标准 JSON，无语法错误）。







```
# ========== 核心配置（复制整段执行） ==========
# 1. 设置绑定模式为lan（允许公网/局域网访问）
openclaw config set gateway.bind "lan"

# 2. 设置监听端口为18789
openclaw config set gateway.port 18789

# 3. 自动生成Token
openclaw config unset gateway.auth.token


放行所有来源访问 Control UI（解决 origin not allowed 报错）
openclaw config set gateway.controlUi.allowedOrigins "*"

# 4. 启用Token认证（必须！防止公网裸奔）
openclaw config set gateway.auth.mode "token"

# 5. 放行Windows防火墙18789端口（管理员权限必做）
New-NetFirewallRule -DisplayName "OpenClaw-18789" -Direction Inbound -Protocol TCP -LocalPort 18789 -Action Allow

# 6. 重启网关使所有配置生效
openclaw gateway restart
```

### 验证配置 & 访问测试

1. **确认认证配置生效**：

```
# 查看认证模式（应为token）
openclaw config get gateway.auth.mode

# 查看自定义 Token（如果用方法1）
openclaw config get gateway.auth.token
```

1. 访问 OpenClaw WebUI

   

   无论用哪种方法，访问时都需要携带 Token，格式如下：

- 本地访问：`http://localhost:18789?token=你设置/查到的Token`
- 公网 / 局域网访问：`http://你的IP:18789?token=你设置/查到的Token`

### 五、补充：完整的公网配置回顾（避免遗漏）

```
# 1. 设置绑定模式为lan（允许公网访问）
openclaw config set gateway.bind "lan"

# 2. 设置监听端口为18789
openclaw config set gateway.port 18789

# 3. 配置自定义Token并启用认证
openclaw config set gateway.auth.token "Claw@18789Token"
openclaw config set gateway.auth.mode "token"

# 4. 放行Windows防火墙端口（管理员身份运行）
New-NetFirewallRule -DisplayName "OpenClaw-18789" -Direction Inbound -Protocol TCP -LocalPort 18789 -Action Allow

# 5. 重启网关
openclaw gateway restart
```

### 总结

1. 核心修正：放弃 `openclaw gateway token generate` 命令，改用 **`openclaw config set gateway.auth.token`** 手动配置 Token；
2. 关键步骤：绑定模式设为 `lan` + 端口设为 `18789` + 配置 Token 并启用认证 + 放行防火墙端口；
3. 访问要求：必须携带 Token 访问 WebUI，无 Token 无法进入，保障公网安全。

如果配置后仍无法访问，执行 `openclaw gateway status` 查看网关是否处于 `running` 状态，有具体状态提示可以告诉我，我帮你定位问题。

## 五、访问测试

配置完成后，用浏览器 / 命令行测试访问：

```
本地访问：http://localhost:18789?token=你生成的Token
公网 / 局域网访问：http://你的IP:18789?token=你生成的Token
```



### 总结

1. 核心修正：端口配置项从 `gateway.listen_port` 改为 **`gateway.port`**，绑定模式用 `gateway.bind "lan"` 替代 `0.0.0.0`；
2. 完整配置：先设 `gateway.bind "lan"` → 再设 `gateway.port 18789` → 重启网关；
3. 安全兜底：必须放行防火墙端口 + 启用 Token 认证，避免公网暴露风险。

如果执行命令后仍有报错，可执行 `openclaw config list gateway` 查看所有网关配置项，确认没有残留的旧配置，我再帮你进一步排查。
