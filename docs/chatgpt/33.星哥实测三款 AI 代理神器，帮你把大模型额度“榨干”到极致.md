## 一、万能兼容的“老大哥”：`New-API`

在个人开发者和中小团队圈子里，`New-API` 几乎是“标配”。它的定位非常明确：**多模型统一网关**，主打一个“啥都能接，接完统一输出”。

### 协议转换，化繁为简

输入支持 API Key 或 OAUTH 授权，输出**强制统一为 OpenAI 兼容格式**。无论你是接国内百炼、硅基流动，还是海外小众模型，只要上游协议选对，填个 API 地址，99.99% 都能秒通。下游业务开发极其丝滑，无需为不同模型写适配层。

### 适合谁？不适合谁？

- ✅ **天选场景**：个人项目、小团队、喜欢“薅免费羊毛”的玩家、上游接口五花八门且变动频繁。
- ❌ **短板**：对纯订阅制/额度池/账号池化管理的支持较弱。如果你手里全是 Cookie/Token 拼车账号，New-API 会让你觉得使不上劲。

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1780763904732.png?tx)

## 二、性能与稳定性的天花板：`gpt-load`

如果你追求**极致的高并发、透明的故障转移和细粒度的团队权限**，这款用 Go 语言手搓的透明代理绝对是首选。它的核心逻辑很简单：把 Key 扔进池子，剩下的交给它。

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1780763806011.png?tx)

### 智能轮询，无感防限流

Key 按分组管理，每组可绑定不同模型与上游地址。请求打入后，自动挑选健康 Key 发送；一旦失败秒切下一个，连续翻车的 Key 会被自动拉入“临时黑名单”，冷却后恢复。整个过程对下游调用方**完全透明**，业务代码一行不用改。

### 弹性架构，轻量到集群无缝切换

- **单机开箱**：默认内置 SQLite，Docker 一行命令直接起飞。
- **集群扩容**：切换 `MySQL + Redis` 架构即可。Follower 节点只需设置 `IS_SLAVE=true`，共享同一数据库，Leader-Follower 架构轻松实现水平扩容，零代码侵入。

### 动静分离 & 权限隔离

静态配置走环境变量（端口、DB 等），动态配置（系统设置、分组策略）全部在 Vue3 打造的 Web 后台完成，支持**热更新，无需重启**。支持全局 Key 与分组级 Key 双认证，不同团队拿不同 Proxy Key，权限严格隔离。配合按天轮转的请求日志，谁调了什么、消耗多少 Token，审计一目了然。

### 性能拉满

零拷贝流式传输、连接池复用、原子操作……懂 Go 的兄弟都懂这些底层优化意味着什么。低延迟、高吞吐，专为重负载场景打造。目前 GitHub 已破 **6000+ Star**，社区活跃，文档齐全。

## 三、订阅额度“拼车”专家：`Sub2API`

如果你手里有大量按订阅制计费的账号（Cookie、Token 或额度池），想精细化榨干每一分余额，`Sub2API` 就是为你量身定制的“调度中枢”。它的数据面板和监控维度做得相当细致。

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1780763842901.png?tx)

### 核心杀手锏：池模式 + 粘性会话

支持将多个订阅账号扔进同一个资源池。下游调用时，自动执行负载均衡与故障转移。更贴心的是**粘性会话**机制，能保证多轮对话始终绑定同一账号，彻底告别“聊着聊着突然失忆”的尴尬。

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1780763861461.png?tx)

### 部署避坑指南（星哥血泪总结）

1. **一键拉起**：官方提供 Docker 部署脚本，配置好 `.env`（PG 密码、JWT 密钥、管理员账号等）即可 `docker-compose up -d`。
2. ⚠️ **Nginx 必填项**：反代配置中必须加上 `underscores_in_headers on;`，否则多账号环境下的粘性会话直接失效！
3. **平台强匹配警告**：创建渠道时，上游平台类型一旦选定无法修改，且必须严格对应接口类型。例如对接 Anthropic 系，必须选 `anthropic-messages`，选错直接连不上。这一点在兼容性上确实不如 New-API 灵活。
