## 部署教程（按需选择）

为方便不同技术基础的玩家，星哥整理了从“零代码”到“全自动化”的三种方案。以下以飞牛NAS系统为例，其他Docker环境逻辑完全一致。

### 方式一：镜像仓库图形化部署（推荐新手）

1. 打开飞牛桌面Docker应用 → 点击左侧【镜像仓库】→ 顶部搜索 `solara-music`。
2. 认准第一个 `1555904/solara-music`，点击下载箭头，弹窗确认即可。
3. 切换至【本地镜像】，找到刚下载的镜像，点击右侧【运行】▶️。
4. 设置容器名称后进入端口映射：本地端口填你闲置的端口（如 `3011`），容器内部端口保持 `3001`。确认防火墙已放行该端口，点击创建，数十秒后容器自动运行。

### 方式二：SSH 命令行一键部署（适合进阶）

打开终端或SSH连接NAS，直接粘贴以下指令（端口号可按需替换）：

```
docker run -d --name solara-music -p 3011:3001 -e NODE_ENV=production -e PORT=3001 --restart unless-stopped 1555904/solara-music:latest
```

💡 *星哥提示：若拉取镜像速度较慢或超时，可在镜像名前添加加速源前缀，例如 `docker.1ms.run/1555904/solara-music:latest`。*



### 方式三：Docker-Compose 编排部署（适合极客/多容器管理）

在Docker的Compose页面新建项目，指定配置文件路径（建议提前在docker目录下建好 `solara` 文件夹），将以下代码粘贴至编辑器：

```
services:
  solara-music:
    image: 1555904/solara-music:latest
    container_name: solara-music
    restart: unless-stopped
    ports:
      - "3011:3001"
    environment:
      - NODE_ENV=production
      - PORT=3001
```

保存并点击部署，系统会自动拉取并启动容器。后续升级、备份、迁移均可通过单文件管理，更适合长期稳定运行。

*(注：原文示例URL中端口写为3111，此处已统一修正为实际映射的3011，避免访问失败。)*
