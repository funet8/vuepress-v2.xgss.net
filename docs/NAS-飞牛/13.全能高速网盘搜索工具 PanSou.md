# NAS 部署指南

Docker 一键上手，零门槛操作。

对于 NAS 用户来说，部署便捷性至关重要，PanSou 提供了 Docker 和源码两种方式，星哥优先推荐 Docker 部署，全程无需复杂配置：

## 1. 直接使用Docker命令

无需单独配置前端，一键启动就能用，适合直接在 NAS 上搭建 web 搜索服务：

```plaintext
docker run -d --name pansou -p 80:80 ghcr.io/fish2018/pansou-web

由于宿主机80端口被占用了，我这边改成82
实际操作
docker run -d --name pansou -p 82:80 ghcr.io/fish2018/pansou-web

```

启动后访问 NAS 的 IP 地址，就能直接使用 web 界面搜索，开箱即用，零配置成本。

如果想自定义前端或集成到其他服务，可部署纯后端版本，通过 API 调用搜索功能：

```plaintext
# 下载配置文件
curl -o docker-compose.yml https://raw.githubusercontent.com/fish2018/pansou/refs/heads/main/docker-compose.yml
# 启动服务
docker-compose up -d
```

默认端口 8888，访问`http://NAS_IP:8888`即可调用 API，支持 POST/GET 两种请求方式，详细参数可参考官方文档。

## 3. 关键配置说明（NAS 优化要点）

- 端口修改：默认 8888，可通过`PORT`环境变量调整，避免端口冲突；
- 代理配置：访问 Telegram 频道需要 SOCKS5 代理，添加`PROXY=socks5://127.0.0.1:1080`（替换为你的代理地址）；
- 缓存挂载：纯后端版建议挂载缓存目录到 NAS 存储，添加`-v /volume1/docker/pansou-cache:/app/cache`，避免容器重启后缓存丢失；
- 频道自定义：默认搜索`tgsearchers3`频道，可通过`CHANNELS`环境变量添加多个频道，用逗号分隔，比如`CHANNELS="tgsearchers3,yunpanxunlei,baiduyun"`。

# API 使用示例：灵活调用，满足多样需求

部署完成后，除了 web 界面，还能通过 API 调用搜索功能，支持自定义关键词、搜索来源、网盘类型等，星哥给大家举个实用示例：

### POST 请求示例（推荐，支持复杂参数）

```json
{
  "kw": "凡人修仙传",
  "channels": ["tgsearchers3", "baiduyun"],
  "conc": 5,
  "refresh": false,
  "res": "merge",
  "src": "all",
  "plugins": ["jikepan", "pan666"],
  "cloud_types": ["baidu", "aliyun", "quark"]
}
```

这个请求会同时从 2 个 TG 频道 + 2 个插件搜索 “凡人修仙传”，并发数 5，仅返回百度、阿里云、夸克三种网盘的结果，结构清晰易处理。

### GET 请求示例（快速测试）

```plaintext
GET /api/search?kw=凡人修仙传&channels=tgsearchers3,baiduyun&cloud_types=baidu,aliyun&res=merge
```

直接在浏览器访问就能返回结果，适合快速测试或简单搜索需求。
