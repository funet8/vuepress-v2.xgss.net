# 星哥带你玩飞牛NAS-11：咪咕视频订阅部署全攻略

## 前言

在家庭影音系统里，NAS 不仅是存储中心，更是内容分发的核心。很多朋友在折腾 I &P  T &V 或视频源时，都会遇到源不稳定、播放卡顿的问题。今天我们来聊一个实用方案——在 NAS 上部署 **咪咕视频订阅源**，实现稳定、持续的影视体验。

## 环境准备

- 一台运行良好的 NAS（飞牛、群晖、威联通或自建 Linux 服务器均可）。
- Docker 环境（推荐，方便管理和升级）。
- 网络条件：保证公网访问或内网穿透，避免源被局限。

## 部署步骤

我这边用飞牛NAS演示

### 1.新建目录

我这边新建的目录是 migu_video

```
/vol2/1000/Docker/migu_video
```

![img](https://imgoss.xgss.net/picgo-tx2025/QQ_1765120230684.png?tx)



### 2.新建Docker compose项目

 在 飞牛NAS 的 Docker 中执行：

![img](https://imgoss.xgss.net/picgo-tx2025/QQ_1765120398689.png?tx)

在docker-compose.yml 填写

```bash

services:
  migu_video:
    image: docker.1ms.run/develop767/migu_video:latest   
    container_name: migu_video            # 容器名称，可自定义
    ports:
      - "1234:1234"                       # 映射端口，左边可自行修改
    environment:
      # 如果不需求蓝光，可只保留mhost和mport
      - muserId=                          # 可选：咪咕帐号ID（不填则匿名访问）
      - mtoken=                           # 可选：咪咕登录令牌
      - mport=1234                        # 容器服务端口，保持与上面一致
      - mhost=http://NAS ip:1234       # 外部访问地址，请替换为你的NAS或服务器IP
      - mrateType=3                       # 清晰度：2=标清，3=高清，4=蓝光(VIP)
    restart: always              # 容器异常退出后自动重启
```

## 常见问题解决方法

### 问题：容器启动后无法访问

**排查步骤**：

1. 检查 NAS 防火墙是否开放了 1234 端口
2. 确认 Docker 容器状态是否为「运行中」
3. 尝试更换端口号（如 8080），避免端口冲突

### 问题：部分频道无法播放

这是由于咪咕视频的版权限制，部分频道仅支持中国大陆 IP 访问。如果你的 NAS 使用了代理，建议关闭或配置国内节点。另外，定期重启容器可以刷新节目源，解决频道失效问题。

## 使用咪咕

订阅地址：IP+端口

打开你常用的播放器，我都用的是potplayer。然后将NAS的IP+端口号输入在配置链接。建议把自动刷新和预览图打开。

 在浏览器或 I P T & V 客户端输入订阅地址，确认频道列表正常加载。
