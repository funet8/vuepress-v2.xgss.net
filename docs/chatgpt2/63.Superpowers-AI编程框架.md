## 核心机制

Superpowers 内置了 15 个硬核技能，但星哥认为，它最绝的设计是拼成了一条 **4步固定链路**。每次执行任务，AI必须老老实实走完这四步：

### 1️⃣ Brainstorm（先问清楚）

AI不能上来就写代码，必须先和你“头脑风暴”，确认需求边界和架构选型，避免方向性错误。

### 2️⃣ Worktree（Git隔离）

强制为当前任务创建独立的 Git 工作区。主线代码绝对安全，随便AI怎么折腾，大不了直接丢弃这个分支。

### 3️⃣ Plan（拆解微任务）

把大需求强制拆解成 2~5 分钟就能完成的“微任务”。化整为零，降低AI出错的概率。

### 4️⃣ Execute（带锁执行）

子代理逐个执行微任务。这里有多重“硬锁”：

- **TDD硬锁**：必须先写失败测试 ➔ 再写实现 ➔ 最后验证通过。杜绝“先写代码后补测试”的陋习。
- **自动审查**：任务与任务之间，自动触发 Code Review，不疲劳、不赶时间。

## 如何安装使用Superpowers 

### 安装Superpowers 

Superpowers 的定位非常清晰：它是 Cursor、Claude Code、CodeBuddy 等工具的 **“方法论插件”**。

安装极其简单，基本都在 30 秒以内：

- **Claude Code**：终端运行 `/plugin install superpowers@claude-plugins-official`，重启即可。
- **CodeBuddy**：打开 Agent 面板 ➔ 插件 ➔ Skill 管理 ➔ 导入 Superpowers 技能文件（或直接把 `SKILL.md` 扔进 `.codebuddy/skills/` 目录）。
- **Cursor / Codex CLI**：直接在插件市场或命令行搜索 `superpowers` 安装。



![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1781339639879.png?tx)



选项

| 选项                                                         | 作用范围               | 配置文件位置                               | 适用场景                                                     |
| ------------------------------------------------------------ | ---------------------- | ------------------------------------------ | ------------------------------------------------------------ |
| **Install for you (user scope)**                             | 你的所有项目全局生效   | `~/.claude/settings.json`                  | ✅ **推荐个人使用**，Superpowers 是个人开发效率工具，全局安装最方便 |
| Install for all collaborators on this repository (project scope) | 当前仓库所有协作者生效 | 项目目录下的 `.claude/settings.json`       | 团队协作项目，需要所有人都使用相同的插件配置                 |
| Install for you, in this repo only (local scope)             | 仅你在当前仓库生效     | 项目目录下的 `.claude/settings.local.json` | 仅当前项目需要使用，不想影响其他项目的配置                   |

直接选择默认高亮的 **`Install for you (user scope)`** 即可

装完后，在终端输入 `/superpowers`，看到 `brainstorm`、`write-plan`、`execute-plan` 等命令亮起，就说明你的 AI 已经穿上“纪律铠甲”了！
