# 用一句话操控手机？这款开源神器把我看傻了



你有没有过这样的体验：想让手机批量处理邮件、自动下载应用，却要在屏幕上点来点去反复操作？想通过语音指令完成复杂任务，却被传统语音助手的 “死板” 限制住？如今，一款名为 mobile-use 的开源 AI 代理工具，彻底打破了这种局限 —— 只需用自然语言说句话、打段文字，手机就能自动完成一连串复杂操作，堪称移动设备的 “智能管家”。

早在大模型尚未普及的年代，小米小爱同学等语音助手就已经让我们见识到了自然语言控制手机的雏形。但受限于技术，当时的功能大多集中在打开应用、设置闹钟等基础操作，面对 “提取 10 条未读邮件并整理成表格”“在应用商店搜索并安装指定游戏” 这类复杂任务时，往往显得力不从心。而 mobile-use 的出现，借助大模型的强大理解能力和智能 UI 交互技术，让自然语言操控手机真正进入了 “全能时代”。



## 一、行业领先的性能，实力碾压同类工具

作为一款开源 AI 代理工具，mobile-use 的硬实力首先体现在权威 benchmark 成绩上。在 AndroidWorld 移动 AI 代理平台基准测试中，它以 74.14% 的通过率拿下开源领域第一，远超第二名 Droidrun 的 63%，更是把 GUI 浏览器、AndroidGen 等同类工具甩在身后。这样的成绩背后，是其对应用界面的精准识别能力和对复杂指令的深度理解能力，确保了操作执行的成功率。

更值得一提的是，它的兼容性覆盖了主流移动设备场景：无论是物理安卓手机、安卓模拟器，还是 macOS 系统下的 iOS 模拟器，都能无缝适配（目前暂不支持物理 iOS 设备）。这意味着无论是开发者测试应用，还是普通用户日常使用，都能找到合适的使用场景。



## 二、五大核心功能，重新定义手机交互方式

mobile-use 之所以能被称为 “逆天工具”，离不开其全面且实用的功能设计，每一项都精准命中用户痛点：

### 1. 自然语言控制：母语交流，无需学习

不用记复杂指令，也不用适配特定句式，用中文直接说出你的需求即可。比如 “打开微信给 XX 发消息说今晚聚餐”“查询最近三天的天气并生成日程”，工具都能精准理解，让手机操作像聊天一样简单。

### 2. UI 感知自动化：智能导航，精准操作

它就像有了 “眼睛” 和 “双手”，能自动识别应用界面的按钮、输入框、列表等元素，规划最优执行路径。哪怕是结构复杂的应用，也能绕过广告弹窗、跳过无关步骤，直达目标功能，完全不用人工干预。

### 3. 数据抓取功能：信息提取，结构化输出

这一功能对办公人士尤其友好。无论是从购物 APP 提取商品价格、从新闻 APP 汇总热点资讯，还是从邮箱导出未读邮件，都能按照你指定的格式（如 JSON、表格）整理输出，省去了手动复制粘贴的繁琐过程。

### 4. 可扩展可定制：适配不同 LLM，满足个性化需求

工具支持灵活配置不同的大语言模型（LLM），无论是 OpenAI 的模型，还是国内的通义千问、文心一言等，都能按需替换。开发者还可以通过自定义配置文件，适配更多特殊场景，让工具的实用性进一步提升。

### 5. 多设备支持：跨平台兼容，场景全覆盖

不管你是用安卓手机办公，还是用 iOS 模拟器做开发测试，mobile-use 都能轻松 hold 住。尤其是对安卓设备的完美支持，无论是物理机还是模拟器，都能实现稳定连接和高效操作。



## 确认你是否已安装 ADB

如果你没安装过 Android SDK / Platform Tools，那 100% 会报这个错。

### 1.安装谷歌Platform Tools

下载 Google 官方 Platform Tools

`https://developer.android.com/studio/releases/platform-tools`

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1769422104636.png?tx)



解压到一个固定目录，例如：

```
D:\mysoft\platform-tools
```

### 2.把 ADB 加入系统 PATH（关键步骤）

#### **临时添加（当前 PowerShell 有效）**

在 PowerShell 中执行：

```
$env:Path += ";D:\mysoft\platform-tools"
```

然后再运行你的命令。

#### 永久添加到系统环境变量

1. Win + R → 输入 `sysdm.cpl`

2. 选择 **高级 → 环境变量**

3. 找到 **Path**

4. 添加：

   ```
   C:\Android\platform-tools
   ```

添加后重新打开 PowerShell。

### 3.验证 ADB 是否可用

在 PowerShell 输入：

```
adb version
```

如果输出类似：

代码

```
Android Debug Bridge version 1.0.41
```

说明成功。

# ✅ **四、确保手机已连接并授权**

运行：

powershell

```
adb devices
```

如果看到：

代码

```
xxxxxx    device
```

说明连接正常。

如果显示 `unauthorized`，请在手机上点“允许 USB 调试”。

# ✅ **五、再次运行你的 mobile-use 命令**

确保 ADB 正常后，再执行：

powershell

```
powershell.exe -ExecutionPolicy Bypass -File mobile-use.ps1 `
  "Open Gmail, find first 3 unread emails, and list their sender and subject line" `
  --output-description "A JSON list of objects, each with 'sender' and 'subject' keys"
```

## 三、简单两步，快速上手使用

对于普通用户来说，mobile-use 的安装和启动门槛并不高，官方提供了 Docker 快速启动和手动安装两种方式，其中 Docker 方式最适合新手：

### 1. 环境准备

首先需要安装 Docker，以及对应设备的辅助工具：安卓设备需安装 Android Debug Bridge（ADB），macOS 系统下的 iOS 模拟器需安装 Xcode。

同时还要确保安装了 uv（Python 包管理器）和 Maestro（设备交互框架）。



### 2. 快速启动（以安卓设备为例）

- 连接安卓设备（开启 USB 调试）或启动安卓模拟器；
- Linux/macOS 用户在终端执行以下命令：

```
chmod +x mobile-use.sh
./mobile-use.sh "Open Gmail, find first 3 unread emails, and list their sender and subject line" \
--output-description "A JSON list of objects, each with 'sender' and 'subject' keys"
```

- Windows 用户在 PowerShell 终端执行：

```
powershell.exe -ExecutionPolicy Bypass -File mobile-use.ps1 `
"Open Gmail, find first 3 unread emails, and list their sender and subject line" `
--output-description "A JSON list of objects, each with 'sender' and 'subject' keys"
```

如果是开发者想要自定义配置，也可以通过克隆仓库、配置环境变量、安装依赖的方式手动启动，支持更灵活的模型适配和功能扩展。





## 四、实战体验：一句话下载游戏，全程自动操作

为了验证工具的实用性，我在安卓模拟器上做了一个简单测试：让工具 “打开游戏中心，下载梦幻西游”。整个过程的表现完全超出预期：

工具首先自动识别设备信息，建立与模拟器的连接，随后启动规划引擎，拆解出 “打开游戏中心→搜索梦幻西游→找到下载按钮→点击下载” 的执行步骤。在进入游戏中心后，面对首页的推广弹窗，它智能识别出 “跳过” 按钮并点击，而非默认安装推荐应用；找到梦幻西游后，自动定位下载按钮并完成操作，全程无需任何手动干预。

从命令下达，到游戏开始下载，整个过程虽然目前还存在执行速度稍慢的问题，但每一步都精准无误，充分体现了其 UI 感知能力和任务规划能力。对于需要重复执行的复杂任务，这种自动化操作能节省大量时间和精力。

## 五、总结：开源工具的潜力，未来值得期待

mobile-use 的出现，不仅让自然语言操控手机从 “基础功能” 升级为 “全能助手”，更让移动设备自动化成为人人可用的工具。无论是普通用户日常处理繁琐操作，还是开发者进行应用测试、数据采集，它都能发挥巨大作用。

当然，作为一个还在初期阶段的开源项目，它目前也存在一些不足：比如物理 iOS 设备暂不支持、复杂任务执行速度较慢等。但随着社区的不断贡献和优化，这些问题有望逐步解决。如果你是技术爱好者、开发者，或者只是想让手机操作更高效的用户，不妨去 GitHub 上 star 支持，亲自体验这款 “逆天工具” 的魅力。

在 AI 赋能万物的时代，mobile-use 用开源的姿态，为移动设备交互打开了新的可能。相信未来，随着大模型技术的不断迭代，自然语言将成为我们与设备交互的主流方式，而这样的开源工具，正是推动这一变革的重要力量。
