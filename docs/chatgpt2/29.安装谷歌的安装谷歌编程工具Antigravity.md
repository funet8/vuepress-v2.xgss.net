## 安装 Antigravity 的前置条件

简单说，一台电脑、能够上谷歌（解决：美国的代-李打开TUN）、谷歌的账号（账号地区在美国，后面会讲到）

### 系统层面的最低要求

根据官方下载页信息 

#### **1. 操作系统要求**

| 平台        | 最低版本要求                   | 备注                                                  |
| ----------- | ------------------------------ | ----------------------------------------------------- |
| **macOS**   | macOS 12（Monterey）及以上     | 仅支持 Apple Silicon，不再支持 Intel                  |
| **Windows** | Windows 10（64 位）及以上      | 支持 x64 / ARM64                                      |
| **Linux**   | glibc ≥ 2.28、glibcxx ≥ 3.4.25 | 典型支持：Ubuntu 20+、Debian 10+、Fedora 36+、RHEL 8+ |

#### **2. 硬件要求**

来自 Antigravity AI 版本说明  [antigravityai.org](https://antigravityai.org/download)：

- **内存：至少 8GB（推荐 16GB）**
- **磁盘空间：至少 2GB 可用空间**
- **CPU：64 位处理器（macOS 支持 Apple Silicon）**

### 网络与地区要求

虽然官方文档没有直接写，但根据实际使用经验与社区反馈（你之前的参考内容也提到过），Antigravity 的授权依赖 Google 服务，因此需要满足：

#### **1. 网络必须为支持地区（如美国）**

- Antigravity **不走系统代理**
- 必须使用 **TUN 模式 + 全局代理** 才能让其流量走代理
- IP 必须显示为 **United States**

#### **2. Google 账号地区必须是美国**

- 如果账号地区是中国/香港，会直接被拒绝授权
- 可通过 Google 官方表单修改地区（需要审核）

### 账号与权限要求

#### **1. Google 账号（必须）**

- 必须能正常登录
- 必须能访问 Google OAuth 授权页面
- 地区必须为美国（非常关键）

#### **2. 允许应用访问网络与文件系统**

- Windows/macOS 需允许应用通过防火墙
- Linux 需确保 glibc 版本满足要求，否则无法启动

## 开发一款APP-活着吗



请为我开发一款名为「活着吗」的移动应用，包含 iOS 和 Android 版本。

核心功能：

1. 用户注册与登录（手机号/邮箱）
2. 个人中心：查看与编辑个人资料、查看打卡历史
3. 紧急联系人管理：添加/编辑/删除 1–3 位联系人
4. 每日打卡：用户每天点击按钮完成打卡
5. 若用户连续两天未打卡，自动向紧急联系人发送提醒（短信或邮件）

技术要求：

- 前端可用 Flutter / React Native
- 后端需实现用户系统、打卡系统、定时任务、通知服务
- 数据库可用 PostgreSQL / MySQL
- 提供完整 API 文档、部署说明与源码
- 敏感信息需加密存储

UI 要求：

- 极简风格，黑白灰主色
- 打卡按钮醒目
- 操作流程简单清晰
