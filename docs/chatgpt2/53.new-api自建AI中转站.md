## 二、Docker 一键部署（NAS/服务器通用）

只要你的设备安装了 Docker 和 Compose 组件，就能直接跑。下面以星哥常用的 NAS 环境为例，全程不到 5 分钟。

### 1.创建目录结构

在 Docker 工作区新建一个 `new-api` 文件夹，内部再建三个子目录，方便后续数据持久化：

```
mkdir -p new-api/mysql_data
mkdir -p new-api/data
mkdir -p new-api/logs
```

文件夹目录作用

```
new-api/
├── mysql_data/   # 数据库文件
├── data/         # 应用配置
└── logs/         # 运行日志
```



### 2. 编写 Compose 配置

打开 Docker 管理面板 → 新建项目 → 路径指向 `new-api` 文件夹 → 粘贴以下代码（星哥已优化注释，开箱即用）：

```yaml
# 下载项目源码
git clone https://github.com/QuantumNous/new-api.git

# 进入项目目录
cd new-api

# 根据需要编辑 docker-compose.yml 文件
# 使用nano编辑器
nano docker-compose.yml
# 或使用vim编辑器
# vim docker-compose.yml
```

> 💡 **星哥提醒**：`SESSION_SECRET`、`CRYPTO_SECRET` 及数据库密码建议替换为自定义强密码，生产环境更安全。确认无误后点击部署，喝杯咖啡的功夫（约2分钟），服务即启动完成。

## 四、环境变量配置

### 必要环境变量

| 变量名              | 说明                                 | 是否必填 |
| ------------------- | ------------------------------------ | -------- |
| `SESSION_SECRET`    | 会话密钥，多机部署必须一致           | **必填** |
| `CRYPTO_SECRET`     | 加密密钥，使用 Redis 时必填          | 条件必填 |
| `SQL_DSN`           | 数据库连接字符串（使用外部数据库时） | 可选     |
| `REDIS_CONN_STRING` | Redis 连接字符串                     | 可选     |

### 生成随机密钥

```
# 生成 SESSION_SECRETopenssl rand -hex 16# 或使用 Linux 命令head -c 16 /dev/urandom | xxd -p
```



## 五、安装常见问题

### 1.无法访问 3000 端口

1. 检查服务器防火墙是否开放 3000 端口
2. 在宝塔面板 **安全** 中放行 3000 端口
3. 检查云服务器安全组是否开放端口

### 2.登录后提示会话失效

确保设置了 `SESSION_SECRET` 环境变量，且值不为空。

### 3.数据如何持久化

使用 Docker 卷映射数据目录：

```
volumes:  - ./data:/data
```

### 4.如何更新版本

```
# 拉取最新镜像docker pull calciumion/new-api:latest# 使用 Docker Compose 重启docker-compose down && docker-compose up -d# 或使用宝塔容器管理重启容器
```

##  七、使用

模型接好了，怎么让其他 AI 软件识别？答案在【令牌管理】。

![img](https://imgoss.xgss.net/picgo-tx2026/QQ_1778745254224.png?tx)

1. 点击左侧【令牌管理】→ 新建令牌 → 填写名称 → 提交。
2. 系统会生成一段连接信息，格式如下：

```json
{
  "_type": "newapi_channel_conn",
  "key": "sk-xxxxxx",
  "url": "http://localhost:3000"
}
```

3. ⚠️ **关键一步**：将 `url` 中的端口 `3000` 替换为你实际映射的端口（如星哥用的 `3000`）。修改后如下：

```json
{
  "_type": "newapi_channel_conn",
  "key": "sk-xxxxxx",
  "url": "我的域名接口"
}
```

现在，这组 `key` + `url` 就是你的 **"AI万能钥匙"**。  
打开任意支持 OpenAI 格式的客户端（如 NextChat、Chatbox、OpenClaw 等），填入自定义 API 地址和 Key，即可自由切换不同大模型。

> 🌰 星哥实测：在终端运行 `openclaw models list`，所有接入的模型瞬间列队待命。切换模型无需改配置，体验极其丝滑。
