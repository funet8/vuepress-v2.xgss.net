## 选择 BillionMail？

市面上大多数邮件平台要么闭源，要么功能残缺。BillionMail 的出现，就是为了打破这些规则：

### 1. 完全开源，拒绝锁定

代码公开透明，没有隐藏费用，没有供应商锁定（Vendor Lock-in）。你想怎么改就怎么改，想部署在哪就部署在哪。

### 2. 无限发送，没有上限

不像 SaaS 平台那样按发送量收费。只要你的服务器带宽和性能扛得住，**想发多少发多少**，彻底解放营销手脚。

### 3. 高级数据分析

别以为自建的就简陋。BillionMail 内置了专业的分析面板：

- 追踪投递率
- 监控打开率
- 统计点击率 让你像专业营销人员一样，用数据驱动决策。

### 4. 隐私优先，数据自有

所有数据都存储在你自己的服务器上。没有第三方跟踪，没有数据泄露给广告商的风险。对于注重合规和隐私的企业来说，这是刚需。

### 5. 专业模板，开箱即用

内置可重复使用的专业营销模板，无需从头设计，快速创建高颜值的邮件内容。

## 如何一键部署？

星哥最关心的就是**部署难不难**。BillionMail 对新手非常友好，提供了两种安装方式。

### 方式一：自动化脚本安装（推荐 ⭐）

这个脚本会自动帮你安装所有依赖环境，包括 Docker，适合懒人操作。

只需在终端执行以下命令：

```
cd /opt && git clone https://github.com/aaPanel/BillionMail && cd BillionMail && bash install.sh
```

### 方式二：Docker 手动安装

如果你已经安装了 Docker 和 docker-compose-plugin，可以手动配置：

1. 克隆项目：

   ```
   cd /opt && git clone https://github.com/aaPanel/BillionMail && cd BillionMail
   ```

2. 初始化环境变量：

   ```
   cp env_init .env
   # 记得根据实际需求修改 .env 文件中的配置
   ```

3. 启动服务：

   ```
   docker compose up -d 
   # 如果提示找不到 compose，试试 docker-compose up -d
   ```

## 常用管理命令

安装完成后，怎么管理？BillionMail 提供了一套简洁的 `bm` 管理脚本：

- **查看帮助**：`bm help`
- **获取默认登录信息**：`bm default` *(忘了密码？跑这个就行)*
- **查看 DNS 记录**：`bm show-record` *(配置域名解析必备)*
- **更新系统**：`bm update`

### 网页邮箱集成

没想到吧？它还集成了经典的 **RoundCube** 网页邮箱！

你可以直接通过 `/roundcube/` 路径访问，方便日常收发邮件和管理联系人，不需要额外搭建邮箱客户端。
