## 为什么传统备份方式

在云原生时代，数据库类型日益丰富，但很多团队的备份方案依然停留在“上古时期”：

- 靠 `Shell + Crontab` 硬扛，脚本散落在各台服务器，维护成本极高；
- 备份成功与否全靠“玄学”，失败无感知，恢复时才发现数据已损坏；
- 存储、压缩、加密、同步上云，每一步都要手动拼接；
- 转向 `PgBackRest` 或 `Barman` 等专业工具，又面临配置晦涩、学习曲线陡峭的困境。

难道就没有一款工具，既能保留底层命令的简洁，又能提供企业级的可视化管控？答案是肯定的。本期主角 **Databasus** 正是为此而生。

## Databasus核心能力

Databasus 最初为 PostgreSQL 设计，如今已全面支持 **MySQL、MariaDB 与 MongoDB**。

**Databasus** 支持备份以下数据库及版本：

- **PostgreSQL**：12、13、14、15、16、17、18
- **MySQL**：5.7、8、9
- **MariaDB**：10、11、12
- **MongoDB**：4.2+、5、6、7、8

它将复杂的备份逻辑封装进现代化的 Web 控制台，核心亮点如下：

### 1️⃣ 灵活精准的定时调度

原生支持小时/日/周/月周期，并完整兼容 Cron 表达式。你可以轻松将备份任务设定在凌晨业务低峰期，无需人工盯守。

### 2️⃣ 自带“数据库体检”机制

不止是备份工具，更是健康管家。系统会定时 Ping 探测数据库连通性，连续失败达到阈值即自动判定宕机，并联动告警通道，将隐患拦截在爆发前。

### 3️⃣ 全场景存储目的地覆盖

本地磁盘只是起点。工具无缝对接 S3 兼容存储、Google Drive、Dropbox、FTP/SFTP、WebDAV 及企业 NAS。数据存哪儿、怎么同步，完全由你掌控。

### 4️⃣ 全渠道实时消息触达

备份成败第一时间推送。已内置 Slack、Discord、Telegram、Email 及自定义 Webhook 集成，支持成功/失败/健康检查异常等多维度告警策略。

### 5️⃣ 企业级安全基线设计

- 采用 `AES-256-GCM` 强加密保护备份包；
- 连接数据库默认仅申请**只读权限**，从架构层面切断“备份工具被入侵导致数据误删”的风险链。

### 6️⃣ 团队协作与操作溯源

支持多用户角色划分（管理员/查看者），内置完整审计日志。谁在什么时间修改了配置、触发了备份，全程可追溯，满足合规要求。

### 1.自动安装脚本（推荐，仅支持 Linux 系统）

该安装脚本可实现以下功能：

✅ 安装 Docker 及 Docker Compose（若未安装）

✅ 部署数据库

✅ 设置系统重启后自动开机自启

```
sudo apt-get install -y curl && \
sudo curl -sSL https://raw.githubusercontent.com/databasus/databasus/refs/heads/main/install-databasus.sh | sudo bash
```

### 2.Docker 命令

只需要输入如下命令

这条单行命令可实现：

✅ 启动数据库服务

✅ 所有数据存放至`./databasus-data`目录

✅ 系统重启后自动重启服务

```
docker run -d \
  --name databasus \
  -p 4005:4005 \
  -v ./databasus-data:/databasus-data \
  --restart unless-stopped \
  databasus/databasus:latest
```

### 3: Docker Compose 命令安装

创建一个文件 `docker-compose.yml` 填入以下内容

```
services:
  databasus:
    container_name: databasus
    image: databasus/databasus:latest
    ports:
      - "4005:4005"
    volumes:
      - ./databasus-data:/databasus-data
    restart: unless-stopped
```

## 恢复数据库

### 1.先安装解压工具

```
# Debian / Ubuntu
apt install zstd -y

# CentOS / RHEL
yum install zstd -y
```

### 2.、一键解压 + 恢复（直接复制用）

先解压，再恢复（最稳妥）

```
# 1. 解压 .zst 文件 → 得到 .sql
zstd -d 备份文件名.sql.zst

# 2. 恢复到 MySQL / MariaDB
mysql -u 用户名 -p 数据库名 < 备份文件名.sql
```

不解压直接恢复（更快）

```
zstd -d 10号机_backup_2026-05-21_03-32-11.sql.zst -c | mysql -u 用户名 -p 数据库名
```

### 3、Databasus 面板内一键恢复

如果你不想敲命令，**直接在 Databasus 网页里点一下就恢复**：

1. 打开 Databasus
2. 进入对应的数据库备份任务
3. 找到这条备份记录
4. 点击 **Restore / 恢复**
5. 选择目标数据库 → 确认
