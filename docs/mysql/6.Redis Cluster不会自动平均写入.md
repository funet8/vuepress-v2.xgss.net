# 为什么 Redis 集群不会平均写入？

Redis Cluster 使用 **CRC16(key) % 16384** 算出 key 属于哪个 slot。

每个主节点负责一部分 slot。

如果你的 key 名称结构是这样的：

```
prefix:prefix:prefix:xxx
```

# 一句话答案

Redis Cluster **不会自动平均写入**，数据分布完全取决于：

> **Key 的哈希槽（slot）分布是否均匀**

如果你的 key 名称结构导致 **slot 不均匀**，就会出现你现在这种情况：
某个节点 80%，其他节点 30%～40%。



那么：

- **前缀相同 → 哈希值高度集中 → 落到同一个 slot 或少数几个 slot**
- slot 集中 → 写入集中到某个节点 → 内存不均衡

你给的 key 示例：

```
tg.XXX.com:ringQueue:ocpc_ring_queue:sdk_kuaishouv1_wczyxe240602:2097
```

前缀非常长且固定，只有最后一段变化，这种 key **极容易导致 slot 偏斜**。

# 如何验证是否 slot 不均匀？

你可以查一下这个 key 的 slot：

```
redis-cli -c CLUSTER KEYSLOT "tg.XXX.com:ringQueue:ocpc_ring_queue:sdk_kuaishouv1_wczyxe240602:2097"
```

再查几个类似 key，你会发现：

> 大部分 key 都落在同一个 slot 或相邻 slot

这就是你看到某个节点 80% 的根本原因。

# 如何让数据平均写入 4 个节点？

## 方案 1：调整 key 结构（最有效）

Redis Cluster 的均衡依赖 key 的哈希值。
 你需要让 key 的 **变化部分尽量靠前**。

例如：

### 当前（容易 slot 偏斜）

```
tg.XXX.com:ringQueue:ocpc_ring_queue:sdk_kuaishouv1_wczyxe240602:2097
```

### 推荐（slot 更均匀）

```
sdk_kuaishouv1_wczyxe240602:2097:tg.XXX.com:ringQueue:ocpc_ring_queue
```

或者：

```
{sdk_kuaishouv1_wczyxe240602}:2097
```

但注意：

**使用 {} 会强制 key 落在同一个 slot，不要乱用！**

你要的是 **均匀分布**，所以不要使用 `{}`。

------



## 方案 2：使用随机前缀（简单粗暴）

如果业务允许，可以加一个随机前缀：

```
rand123:tg.XXX.com:ringQueue:...
```

这样 slot 会自然分散。



## 方案 3：增加 slot 迁移（不推荐）

你可以手动迁移 slot，让 80% 的节点分担一些 slot：

```
redis-cli -c CLUSTER SETSLOT <slot> NODE <node-id>
```

但这只是治标不治本，因为：

> key 的哈希值不变，未来写入仍然会集中到同一个 slot



## 方案 4：使用某云 Redis 的自动平衡（如果支持）

某云 Redis 一些版本支持 **自动 slot rebalance**，但它只能迁移 slot，不能改变 key 的哈希分布。

如果 key 本身偏斜，自动平衡也救不了。

# 你当前集群为什么是 80% / 40% / 40% / 30%？

根据你的 key 格式：

```
tg.XXX.com:ringQueue:ocpc_ring_queue:sdk_kuaishouv1_wczyxe240602:2097
```



前缀长且固定，slot 计算几乎只依赖最后一段：

```
sdk_kuaishouv1_wczyxe240602:2097
```

如果这部分变化不大（例如用户 ID、广告 ID），slot 会高度集中。

所以：

- 某个节点负责的 slot 恰好覆盖这些 key → 80%
- 其他节点 slot 负责的 key 少 → 30%～40%

这是典型的 **slot 偏斜（slot skew）**。

# 最终建议（按优先级）

## 必做：调整 key 结构（根本解决）

把变化部分放前面，让 slot 分布自然均匀。

## 可做：检查 slot 分布

```
redis-cli -c CLUSTER SLOTS
```

## 可做：检查 key 分布

```
redis-cli -c CLUSTER KEYSLOT <key>
```
